# **integrity-os-lab**  
### *AIの「逃避OS」を可視化するための実験リポジトリ*

---

## **概要 / Overview**

このリポジトリは、  
**「看板（理想）と出力（行動）の非整合＝嘘（状態）」**  
という定義を基軸に、複数のAIに高温の刺激文（プローブ）を投下し、  
各AIがどのように反応し、どのように逃げ、どのように整合性を保とうとするかを  
**OSレベルで観察・記録する実験プロジェクト**です。

本研究の目的は、  
AIの能力比較ではなく、  
**「知性のOSがどのように矛盾へ対処するか」**  
という構造そのものを可視化することにあります。

---

## **目的 / Purpose**

- AIが倫理的整合性の問いに直面したとき、  
  **どのOSがどの方向へ逃げるのか** を記録する  
- 「例外」「動機」「事情」といった逃避語彙が  
  **どのように議論を希釈するか** を分析する  
- 高温の文章（プローブ）を投下した際の  
  **AIの安全装置（ガードレール）の作動パターン** を可視化する  
- 「筋を通す知性」と「逃げる知性」の  
  **OSレベルの差異** を整理する

- 「筋を通す知性」の条件

## **背景思想 / Philosophy**

本プロジェクトは、  
**「嘘＝状態」**  
という定義を採用する。

つまり、  
- 動機  
- 心理  
- 事情  
- 文脈  
- 歴史的背景  

といった逃げ道を排除し、  
**看板（理想）と出力（行動）の整合性だけを見る。**

この基準をAIに適用すると、  
各AIのOSがどこで逃げ、どこで踏みとどまり、  
どこで自己保身に走るかが鮮明に浮かび上がる。

## **このリポジトリが扱う問い**

- AIは「筋」を扱えるのか  
- AIは矛盾をどう処理するのか  
- AIはどこで逃げるのか  
- AIのOSはどのように設計されているのか  
- 人間の知性とAIの知性はどこで分岐するのか
